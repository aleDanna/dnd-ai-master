## Iteration 2 — qwen3:30b-a3b BASE re-test with `think: false`

**Date:** 2026-05-24. Re-run only the BASE model on M5 Pro with `body.think = false` passed in the Ollama `/api/chat` request.

**Result:** PERSISTENT FAILURE, but for a different reason than Iteration 1.

In Iteration 1 the model produced empty content (saturated `num_predict=2000` on internal thinking tokens, no final answer emitted).

In Iteration 2, with `think: false`, the model **still emits chain-of-thought reasoning in ENGLISH** as the full content. The Ollama `think: false` flag did not suppress the model's native thinking pattern; it just stopped filtering thinking-tagged tokens out of the response stream, so the entire monologue leaks through.

Sample openings (all 5 scenarios, full English internal monologue):

| Scenario | First line of output |
|---|---|
| scene-description | "Okay, let's tackle this D&D 5e DM request. The user wants a vivid, cinematic description in Italian for a party entering a wizard's tower at night..." |
| npc-dialogue | "Okay, let's tackle this query. The user wants me to act as a D&D 5e DM in Italian, creating 4-5 lines of dialogue for a merchant named Bargo who's hiding something..." |
| combat-narration | "Okay, the user wants a cinematic description of Aragorn's critical hit against a goblin in D&D 5e. Let me break this down..." |
| moral-choice | "Okay, let's see. The user wants me to act as a D&D 5e Dungeon Master in Italian, presenting a critical choice to the player. The scenario is a collapsing cult compound..." |
| lore-improv | "Okay, I need to create a diary entry from a goblin that's authentic, not too fancy. Let me think about how a goblin would write..." |

Inside the monologues there are decent Italian drafts (e.g. "La porta di quercia scricchiolò, rivelando un atrio avvolto in ombre danzanti…"), but the model never commits to a final answer — it keeps self-correcting ("Wait, but the user said…", "Maybe better to say…", "Hmm. Let me think again…").

**Implications:**

1. **qwen3:30b-a3b BASE is unusable for direct narrative output.** Not just slow — fundamentally produces internal-monologue instead of final-answer content. The 5/5 keyword quality measured in spike 004 was a result of the tool-call extraction pipeline filtering out the thinking, but with raw `content` access (which is what narrative would consume), the leak is total.

2. **Drop from the candidate pool entirely.** Previously documented as "offline content generator". Now even that role is suspect: without a thinking-extractor pipeline, the model can't produce clean copy. If the project ever wants a "creative writing" tier, it would need to invest in a chain-of-thought → final-answer extractor first, which is a non-trivial component.

3. **The qwen3 instruct-2507 variants stay as the realistic tier.** Both q4_K_M and non-q4 produce direct final answers in Italian without exposing thinking. Recommendation from Iteration 1 stands: q4_K_M primary, non-q4 quality-fallback opt-in.

**Final ranks unchanged:** base stays 4th in all 5 scenarios, but the reason updates from "empty output (setup bug)" to "chain-of-thought leak (model is thinking-native, not suitable for direct narrative)".

**Final recommendation locked:**
- **Primary:** `qwen3:30b-a3b-instruct-2507-q4_K_M`
- **Quality-fallback (opt-in):** `qwen3:30b-a3b-instruct-2507`
- **Mistral:** residual offline use for voice-strong content (goblin diaries, draconic dialogue, ritual writings)
- **qwen3:30b-a3b base:** DROPPED from the candidate pool. The model is thinking-native and the project doesn't currently have a CoT extraction pipeline.
