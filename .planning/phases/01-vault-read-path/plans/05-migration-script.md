# Plan 05: Handbook + Lore Migration Script

**Phase:** 01-vault-read-path
**Status:** Pending
**Depends on:** 01-vault-path-safety (uses `VAULT_ROOT`)
**Estimated diff size:** ~280 LOC source + ~90 LOC tests / 4 files

## Goal

Ship `scripts/migrate-handbook-to-vault.ts` — a one-shot, idempotent CLI that splits `data/master_handbook.md` + `data/master_world_lore.md` into per-H2 vault files under `data/vault/handbook/{craft,lore}/` and generates `data/vault/handbook/index.md` (TOC) + `data/vault/tools/index.md` (compact tool description per spike 002 lenient protocol) + per-tool stubs.

Idempotent re-run yields byte-identical output (verified by test). The generated `data/vault/` directory IS committed to git in this PR — it becomes the runtime source of truth. The two source markdowns (`data/master_*.md`) stay in place; the baked path still consumes them via `getMasterHandbook` / `getMasterWorldLore`.

Scaffold (empty + `.gitkeep`) the placeholder folders for future Phase-02+ content: `data/vault/handbook/{spells,monsters,items,rules,classes}/.gitkeep`. The vault `read_vault_multi('handbook/spells/fireball.md')` for a missing file returns the standard "file not found" marker — the model handles this gracefully per spike 009. Pre-creating the directories prevents `ENOENT`-on-`readdir` surprises on `list_vault` calls.

## Requirements satisfied

- **REQ-001** Vault is filesystem-only markdown — script produces only markdown files.
- **REQ-002** Static knowledge is path-deterministic: `/handbook/<category>/<id>.md`.
- **REQ-012** Lenient discovery: `/tools/index.md` is the single discovery entry point, generated by this script.

## Tasks

1. **Create `scripts/migrate-handbook-to-vault.ts`.** Structure:

   - Top of file: `#!/usr/bin/env tsx` shebang (consistent with `scripts/build-rag-index.ts`).
   - Constants:
     - `const HANDBOOK_SRC = path.resolve(process.cwd(), 'data/master_handbook.md');`
     - `const LORE_SRC = path.resolve(process.cwd(), 'data/master_world_lore.md');`
     - `const VAULT_OUT = path.resolve(process.cwd(), 'data/vault');`
   - **Parsing.** Function `parseH2Sections(markdown: string): { number: number; title: string; slug: string; body: string }[]`:
     - Split by `/^## (\d+)\. (.+)$/m` (the actual heading shape verified by `grep -n "^##" data/master_handbook.md`).
     - For each H2, the body extends from after the heading line up to (but excluding) the next H2 heading line OR the end of the document.
     - `slug` = `title.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '')`. E.g. `"Your Role" -> "your-role"`, `"Pacing & Narration" -> "pacing-narration"`, `"The Multiverse (Cosmology)" -> "the-multiverse-cosmology"`.
     - **Manual override map** for cleaner slugs that match RESEARCH.md §3 canonical taxonomy (e.g. `"Your Role"` → `role`, `"Knowing the Player"` → `knowing-the-player`, `"The Multiverse (Cosmology)"` → `cosmology`). Encode this as a `MANUAL_SLUGS: Record<string, string>` lookup keyed on the H2 title; fall back to the auto-slug if absent. List of overrides:
       ```
       const MANUAL_SLUGS: Record<string, string> = {
         // craft (handbook)
         'Your Role': 'role',
         'Knowing the Player': 'knowing-the-player',
         'Pacing & Narration': 'pacing',
         'Resolving Outcomes': 'resolving-outcomes',
         'Social Interaction': 'social',
         'Exploration': 'exploration',
         'Combat': 'combat',
         'Improvising': 'improvising',
         'Death and Consequences': 'death',
         // craft (extras not in RESEARCH.md's 9 but present in master_handbook.md H2 grep)
         'Character Advancement': 'character-advancement',
         'NPCs': 'npcs',
         'Common Pitfalls (Avoid These)': 'common-pitfalls',
         // lore (world)
         'The Multiverse (Cosmology)': 'cosmology',
         'Magic in the World': 'magic',
         'Deities and Religion': 'deities',
         'Cultures, Factions, Settlements': 'cultures',
         'Campaign Frames': 'campaign-frames',
         'Common Tropes and Hooks': 'tropes',
         'Rewards and Gratification (CRITICAL)': 'rewards',
         'When the Player Asks About the World': 'world-questions',
       };
       ```
       **Note:** RESEARCH.md listed 9 craft H2s; the actual file contains 12 (the H2 grep above reveals `Character Advancement`, `NPCs`, `Common Pitfalls`). The script SHIPS ALL H2s — under-shipping the migration to match the research's incomplete enumeration would silently drop content. Similarly the lore file has 8 H2s (research counted 7; the 8th is `When the Player Asks About the World`). Migration ships all 12 craft + 8 lore = 20 H2 files.
   - **Frontmatter.** Each generated H2 file gets a YAML frontmatter:
     ```
     ---
     id: <slug>
     category: <craft|lore>
     source: master_handbook.md|master_world_lore.md
     h2_number: <n>
     h2_title: "<original title>"
     ---
     ```
     Followed by a blank line, then the H2 title rendered as `# {title}`, then the original body (preserves H3+ subheadings verbatim).
   - **Write step:**
     - `mkdirSync(path.join(VAULT_OUT, 'handbook/craft'), { recursive: true });`
     - For each parsed craft section, write `craft/<slug>.md` with frontmatter + body. Same for lore (`lore/<slug>.md`).
     - **Idempotency rule.** Read the existing file at the target path (if any) and compare byte-for-byte to the proposed output. ONLY write if different — this both makes re-runs effectively no-ops and produces clean `git status` output when nothing changed.

2. **Generate `data/vault/handbook/index.md`** — TOC pointing to all generated files. Format:

   ```
   ## Craft (DM technique)

   - [Role](./craft/role.md) — your role as the DM
   - [Knowing the Player](./craft/knowing-the-player.md)
   ...

   ## Lore (worldbuilding)

   - [Cosmology](./lore/cosmology.md)
   ...

   ## Reserved (Phase 02+)

   - `./spells/<id>.md` — spell catalog (empty in Phase 01)
   - `./monsters/<id>.md` — monster catalog (empty in Phase 01)
   - `./items/<id>.md`
   - `./rules/<topic>.md`
   - `./classes/<class>.md`
   ```
   The link text for each TOC item is the original H2 title, NOT the slug.
