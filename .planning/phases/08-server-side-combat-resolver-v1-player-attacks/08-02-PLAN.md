<objective>
Add NARRATION-ONLY mode to the vault tool loop and D-07 directive suppression to the turn
directive — the two independent guards that PREVENT DOUBLE-APPLY (the headline integrity
control of REQ-039). `loop.ts` gains a `suppressCombatMutations` flag that drops the LLM's
combat-event `apply_event` calls (only `ENCOUNTER_EVENT_TYPES`) on resolution turns; the
server already emitted the authoritative events. `turn-directive.ts` gains a `serverResolved`
flag that skips the player-side "resolve" directive (the 07-05 re-ask-breaker) so the prompt
doesn't actively instruct the model to emit the very events we drop — belt-and-suspenders.

Purpose: Without these two guards, the server emits `monster_hp_change(-9)` AND the LLM also
emits it → the monster takes 18 damage and the turn double-advances (RESEARCH Pitfall 3, the
core risk). This plan is independent of the resolver (different files) so it runs in Wave 1
alongside Plan 01; Plan 03 wires both into the route.

Output: `loop.ts` + `loop.test.ts` (narration-only drop + non-combat-still-dispatches +
flag-off regression); `turn-directive.ts` + `turn-directive.test.ts` (D-07 suppression +
regression).
</objective>
