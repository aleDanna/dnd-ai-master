Wire the pure opener (10-01) into the vault branch of the turn route so the SERVER opens an encounter on combat intent — independent of the local model emitting apply_event. This is the REQ-045 delivery: detect combat intent → openEncounter → dispatch combat_start + monster_spawn + initiative_set with sessionId → the existing v1/v2 gates re-read events.md and see the opened encounter.

Purpose: close the "encounters silently never open" root cause (qwen3 tool_calls=0). REQ-047 invariant: the gate is ordered AFTER isRollResult, and the opener emits NO damage on the opening turn — only combat_start/monster_spawn/initiative_set — so damage still only lands after the damage roll via the untouched v1 two-step.
Output: the route hook + a headless integration test (the route's first end-to-end proof, per 09 deferred-items recommendation).
