<objective>
Create the pure, server-authoritative encounter-opener function `runEncounterOpener` that turns a combat-intent into the opening combat events: a `monster_spawn` ({id,name,hpMax}) for the detected monster and an `initiative_set` whose order interleaves every PC and the monster. This is the deterministic core of REQ-045 (server selects monster + rolls initiative) and enforces the REQ-047 invariant (no damage on the opening turn).

The function is PURE: it receives the session snapshot, the resolved monster name, and a `bestiaryLookup` callback (dependency-injected so the statblock reader from 10-02 plugs in, and so the future option-A monster-selection path can swap in without touching this code per CONTEXT D-02). It performs no I/O and dispatches nothing — the route (10-03) wires it to `dispatchVaultTool` with campaignId + sessionId.

Purpose: Isolate the opener's branching/initiative logic behind tests so the route stays thin and the bestiary path (10-02) and dispatch path (10-03) can be verified independently.
Output: `src/app/api/sessions/[id]/turn/encounter-opener.ts` + its test file.
</objective>
