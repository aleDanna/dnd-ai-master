<objective>
Wire EncounterState.turnOrder-driven turn handoff into the turn route's vault branch.

Purpose: Without this plan, the turn route ignores the combat state and falls through to detectAddressee for every vault turn — the master's turn_advance events advance currentIdx in the combat.md view but the session never hands control to the correct player. This plan makes the route read the live EncounterState and derive handoff from turnOrder[currentIdx], while preserving the existing detectAddressee fallback for non-combat turns.

This is the risky core of D2. Non-combat multiplayer handoff MUST be unchanged — verified by a dedicated regression test.

Output: Modified turn route (vault branch transaction block); new test file covering combat interleaving (PC actor → handoff, monster actor → no handoff, inactive encounter → fallback) AND non-combat regression (session without active encounter → identical behavior as before).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| EncounterState → route handoff decision | EncounterState is read from combat.md (filesystem, campaign-scoped). actorId values are master-invented strings or PC UUIDs from the DB party list. The party membership check (party.some(p => p.id === actorId)) is the trust gate: only known PC UUIDs trigger a handoff; unknown strings (monster ids) do not. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03-01 | Elevation of Privilege | turnOrder actorId used to set currentPlayerCharacterId | mitigate | actorId is only used to set cpcId if it matches a known party member UUID from the Postgres characters table. Monster ids (master-invented strings) never match party UUIDs, so they cannot trigger a player handoff. The party.some() check is the gate. |
| T-07-03-02 | Denial of Service | Combat interleaving throws, breaks non-combat sessions | mitigate | Entire block is wrapped in try/catch; any exception falls through to the existing detectAddressee path. Non-combat sessions are unaffected by a bug in the new code. The regression test suite enforces this. |
| T-07-03-03 | Tampering | combat.md read injecting malicious actorId | accept | actorId is only used in party.some() membership check — it is never interpolated into SQL, shell commands, or other sensitive contexts. Even a crafted actorId string cannot escalate beyond triggering or skipping a turn handoff. |
