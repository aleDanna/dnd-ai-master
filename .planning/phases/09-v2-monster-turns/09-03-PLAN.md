Implement the SRD bestiary attack-prose parser (D-04) as an ISOLATED module `src/app/api/sessions/[id]/turn/monster-bestiary.ts`. The loop (09-04) calls it FIRST in the 3-level fallback, but a `null` return cleanly cedes to the CR table.
Output: `parseFirstAttackFromProse` + `getBestiaryAttackStats`, slug-safe and ReDoS-safe, unit-tested against real bestiary prose.
