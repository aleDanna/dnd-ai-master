## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cr value → CR table lookup | a cr (from a server-controlled spawn event) crosses into a numeric table lookup; malformed values must not crash |
| livePcIds / pcAcById → combat math | PC ids + AC originate from server state (turnOrder + Postgres AC), never from player input directing the monster |
| damageDice → rollDamage | a dice string crosses into the roller; must be a rollDamage-accepted shape from the controlled table/parse, never attacker-chosen |
| randomness seam | non-determinism is a testability + audit risk; all draws must route through one injected Rng |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | DoS | getMonsterAttackStats cr lookup | mitigate | cr validated (`Number.isFinite` + `>= 0`) before lookup; NaN/Infinity/negative/out-of-range fall back to the named-constant default — never throws (ASVS V5, RESEARCH Security Domain). |
| T-09-05 | Tampering | monster attack stats | mitigate | attackBonus/damageDice derive only from the CR table, bestiary parse, or server constants — never from player input. Target ids/AC come from server-resolved EncounterState/Postgres (ASVS V4). |
| T-09-06 | DoS | randomness seam | mitigate | All draws via injected `Rng`; no `Math.random` → deterministic, testable, no hidden nondeterminism. Empty livePcIds returns null defensively, never throws (mirrors v1 D-10 edge contract). |
| T-09-07 | Tampering | HP underflow | mitigate | resolveMonsterTurn emits only the negative `delta`; the `hp_change` reducer clamps `max(0, hp+delta)` (existing) — this function never drives HP negative (symmetric with v1 T-08-02). |
