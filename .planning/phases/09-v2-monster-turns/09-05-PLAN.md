<objective>
Wire the D-08 schema change into the LLM surface and implement the D-16 directive suppression:

- **D-08 (LLM surface):** update the `apply_event` payload description in `tools.ts` so the `monster_spawn` clause advertises `cr?:number`, and instruct the LLM (in the combat-lifecycle / monster_spawn guidance) to include `cr` as a difficulty hint for custom monsters. The PROMPT-side instruction must NOT break REQ-022 (the system prompt is a pure, byte-stable function — see note below on where the instruction lives).
- **D-16 (suppression):** add a `monsterResolved?: boolean` flag to `TurnDirectiveOpts` (mirroring Phase 08's `serverResolved`) and gate the combat re-ask directives on it, so that when the SERVER has resolved the monster turns (loop ran), the directive does NOT tell the model to emit `hp_change`/`turn_advance`/combat events the loop already emitted — analogous to v1's D-07 player-side suppression.

Purpose: D-08 needs the LLM to actually SEND `cr` for custom monsters (otherwise the smoke-critical D-05 path never gets a cr to map). D-16 prevents conflicting instructions on a server-resolved monster turn (RESEARCH Pattern 7 / Pitfall 3). This plan is Wave 2 but touches DIFFERENT files than 09-04 (tools/prompt/directive vs the resolver module), so it runs in parallel with 09-04.
Output: the LLM is told about + asked for `cr`, and `buildTurnDirective` suppresses combat re-asks on a server-resolved monster turn while staying byte-identical when the flag is absent.
</objective>
